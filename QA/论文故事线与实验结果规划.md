# 论文故事线与实验结果规划

## 1. 先给结论

主故事不要再写成“我们做了一个很大的多任务系统”，而要写成：

```text
真实多通道阵列上的声源方位定位
-> 传统空间扫描基线可解释
-> RealMAN 官方 baseline 可对标
-> 我们用 8ch ring2 子阵列做可学习空间编码 + TCN
-> 再往上扩展到 slot / future / trend / 设备验证
```

这条线更稳，也更像论文，而不是 demo。

## 2. 必须保留的基线

### 2.1 传统基线

`SRP-PHAT peak` 要保留。它不是“花活”，而是传统声源定位里很标准的可解释下限。  
在论文里它的作用是：

- 证明当前模型至少比传统空间扫描更强
- 证明 SRP 空间谱不是摆设
- 作为所有学习型方法的传统参照系

### 2.2 官方基线

RealMAN 官方定位 baseline 主要是：

- `CRNN`
- `IPDnet`

它们负责告诉读者：在 RealMAN 这个真实数据集上，当前主流深度方法能做到什么量级。

因此完整基线链应该写成：

```text
SRP-PHAT peak
  -> RealMAN CRNN / IPDnet
  -> 我们的 8ch ring2 方法
```

## 3. 当前 8ch 消融主线

当前最该做的是 8ch 消融，而不是先恢复多任务。

### 3.1 四个最小消融

已经补好的配置：

- `realman_ring2_ablation_logmel`
- `realman_ring2_ablation_logmel_ipd`
- `realman_ring2_ablation_logmel_srp`
- `realman_ring2_ablation_logmel_ipd_srp`

对应的核心问题是：

| 配置 | 要回答的问题 |
| --- | --- |
| logmel only | 只看语音内容，定位能到什么程度 |
| logmel + IPD | 加相位差后，空间信息能提升多少 |
| logmel + SRP | 显式空间谱是否足够强 |
| logmel + IPD + SRP | 两种空间特征是否互补 |

VAD 先作为辅助输入，不作为主消融对象。

### 3.2 当前主结果

当前定位主结果应该固定成：

```text
RealMAN ring2_8ch
current heatmap localization
MAE / ACC@5
static / moving 分组
```

最新已验证结果是：

- all: `4.49° / 82.25%`
- static: `2.59° / 87.74%`
- moving: `6.84° / 75.44%`

这个量级已经说明路线是成立的。

## 4. 文献故事怎么讲

建议三层叙事：

### 4.1 第一层：传统方法能做，但不够

SRP-PHAT / TDOA 是可解释起点，但对真实场景里的动态、噪声和短窗变化不够稳定。

### 4.2 第二层：RealMAN 给了真实阵列和真实数据

RealMAN 的价值不只是“数据多”，而是：

- 真阵列
- 真噪声
- 真移动源
- 真标注

它让声源定位从仿真走到真实世界。

### 4.3 第三层：我们做的是可学习的空间时序融合

我们不是替代传统方法，而是把：

```text
logmel + IPD + SRP + VAD
```

作为可解释输入，再用 TCN 做短窗时序融合，输出当前方位 heatmap。

## 5. 现在最该跑的实验顺序

按这个顺序：

1. `SRP-PHAT peak` baseline
2. `logmel only`
3. `logmel + IPD`
4. `logmel + SRP`
5. `logmel + IPD + SRP`
6. 更长的 8ch 主结果训练
7. moving 标签对齐检查

## 6. 论文里现在可以稳定写的句子

可以写：

```text
我们在 RealMAN ring2_8ch 子阵列上，以 SRP-PHAT 作为传统可解释基线，并与 RealMAN 官方 CRNN/IPDnet 基线对照，研究 logmel/IPD/SRP 的互补性以及 TCN 时序融合对当前方位定位的提升。
```

不要写：

```text
我们完整替代了所有传统和官方方法。
```

## 7. 目前的风险点

1. moving 仍然弱于 static，需要查标签和窗口对齐。
2. slot/count/future 现在不能作为主结论。
3. 8ch 消融还没正式跑完。
4. 32ch 全量还没上线，现阶段不要把它写成已完成主结果。

```text
先用 SRP-PHAT 作为传统基线，用 RealMAN 官方 CRNN/IPDnet 做对标，再用 8ch ring2 的 logmel/IPD/SRP 消融证明我们的 TCN 空间融合是有效的。
```
