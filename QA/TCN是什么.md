# TCN 是什么

## Q1：TCN 是什么？

TCN 的全称是 `Temporal Convolutional Network`，中文可以叫“时间卷积网络”。

它本质上是用 `1D CNN` 来做时间序列建模。  
如果 RNN 是按时间一步一步递推：

```text
x1 -> x2 -> x3 -> x4 -> x5
```

那么 TCN 是在时间轴上做卷积：

```text
[x1, x2, x3] -> 卷积 -> 时间特征
[x2, x3, x4] -> 卷积 -> 时间特征
[x3, x4, x5] -> 卷积 -> 时间特征
```

所以可以简单理解为：

```text
TCN = 用 CNN 的方式处理时间序列
```

## Q2：它和普通 CNN 有什么区别？

普通 CNN 常用于图像，卷积核在二维空间上滑动：

```text
height x width
```

TCN 用在序列上，卷积核主要沿时间轴滑动：

```text
time x feature
```

比如音频特征可以表示成：

```text
128 帧 x 64 维特征
```

TCN 就是在 `128` 个时间帧上做一维卷积，学习前后帧之间的动态关系。

## Q3：它和 RNN 有什么区别？

RNN 的核心是递归状态：

```text
h_t = f(x_t, h_{t-1})
```

它必须按时间顺序一步一步算。

TCN 不用递归状态，而是用卷积块一次性处理一段序列。  
因此 TCN 通常有几个优势：

- 训练更容易并行
- 梯度传播更稳定
- 感受野可以明确控制
- 推理延迟更容易计算
- 工程实现比复杂 RNN/Transformer 更直接

但 RNN 的优势是天然带有“状态传递”形式，适合非常长的连续流式记忆。  
TCN 更适合固定窗口或中短时程的序列建模。

## Q4：TCN 的两个核心技术是什么？

### 1. 因果卷积

因果卷积的意思是：预测当前时刻时，只能看当前和过去，不能看未来。

例如预测 `t` 时刻：

```text
可以使用：t, t-1, t-2, t-3
不能使用：t+1, t+2
```

这对实时音频系统很重要，因为设备运行时不可能提前知道未来声音。

### 2. 空洞卷积

普通卷积只能看到很近的时间范围。  
空洞卷积通过隔点采样扩大感受野。

例如卷积核大小为 `3`：

```text
dilation = 1:  t, t-1, t-2
dilation = 2:  t, t-2, t-4
dilation = 4:  t, t-4, t-8
dilation = 8:  t, t-8, t-16
```

这样不用堆很多层，也能看到较长历史。

## Q5：TCN 的典型结构是什么？

一个常见 TCN block 大致是：

```text
输入
  -> 因果空洞卷积
  -> 激活函数
  -> dropout / normalization
  -> 因果空洞卷积
  -> 激活函数
  -> dropout / normalization
  -> 残差连接
输出
```

其中残差连接类似 ResNet：

```text
output = F(x) + x
```

这样可以让网络堆得更深，同时训练更稳定。

## Q6：为什么当前声源定位项目适合用 TCN？

当前项目的输入是帧级音频特征，例如：

- `logmel`
- `IPD`
- `SRP`
- `VAD history`

这些特征天然是：

```text
time x feature
```

当前任务也不是只看一帧，而是要根据一段历史判断：

- 当前有几个声源
- 声源在哪个方向
- 多个声源的槽位是否稳定
- 声源是否在移动
- 未来短窗趋势怎么变化

这类问题正好需要短中时程的时间上下文。

TCN 适合当前项目，主要因为：

- 能处理帧级音频序列
- 可以用因果卷积支持实时推理
- 用空洞卷积覆盖 `1.28s` 左右历史窗口比较自然
- 比 RNN 更容易并行训练
- 比 Transformer 更轻，第一阶段工程风险更低
- 和 `IPD/SRP/logmel` 这类特征很容易结合

## Q7：TCN 能解决所有问题吗？

不能。

TCN 主要负责学习短中时程的动态表征。  
但它不会自动解决这些问题：

- 多说话人身份连续性
- 两个声源交叉后的 slot 交换
- 长时间轨迹记忆
- 多源 assignment ambiguity
- 真实设备阵列几何泛化

这些仍然需要配合：

- slot matching
- Hungarian / PIT
- Kalman 或其他轻量跟踪器
- 麦克风 dropout
- 子阵列随机采样
- 真实数据增强

因此更准确的理解是：

```text
TCN 负责短中时程动态建模；
assignment 和 tracking 机制负责多源身份连续性；
RealMAN 子阵列训练负责真实设备泛化。
```

## Q8：一句话总结

TCN 是一种用一维因果空洞卷积处理时间序列的网络。  
它可以看作“面向时间序列的 CNN”，适合当前声源定位项目中 `历史音频特征 -> 当前状态 + 短窗未来趋势` 的建模需求。
