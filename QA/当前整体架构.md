# 当前整体架构

## 1. 当前一句话结论

当前系统已经从“大而全的多任务跟踪模型”收敛为一个更现实的两层架构：

```text
主实验闭环：RealMAN ring2_8ch 当前方位定位
扩展展示层：count / slot / future / trend / 可视化 / 系统链路
```

现阶段论文和申报书最应该依赖的是第一层：**当前声源方位定位**。  
它能和 RealMAN baseline 的 `MAE [deg]`、`ACC(5deg)` 放在同一类指标下讨论。

## 2. 数据输入

当前使用 RealMAN 的 `ring2_8ch` 子阵列：

```yaml
channel_ids: [9, 10, 11, 12, 13, 14, 15, 16]
array_radius_m: 0.06
```

官方 baseline 代码中 RealMAN 32ch 阵列定义为：

```text
R = 0.03
CH9-CH16 = 第二圈 8 麦 = R * 2 = 0.06 m
```

所以当前几何假设和 `CH9-CH16` 是匹配的。  
但注意：官方 CRNN/IPDnet 默认评估通道不是这组，因此当前结果应称为 **RealMAN ring2_8ch 子阵列结果**，不能直接说严格复现官方默认 baseline。

模型输入：

```text
waveform: [B, 8, samples]
history_frames = 128
future_frames = 32
hop_length = 160
sample_rate = 16000
```

当前 localization-only 训练主要看历史约 `1.28 s`，预测当前方位。

## 3. 特征前端

特征前端位于：

[stft.py](/c:/Users/haoming%20lan/Desktop/dc/src/uca8/features/stft.py)

从 8ch waveform 中提取：

| 特征 | 形状 | 作用 |
| --- | --- | --- |
| `logmel_ref` | `[B, 1, T, 64]` | 参考麦克风频谱内容 |
| `logmel_rms` | `[B, 1, T, 64]` | 多通道整体能量频谱 |
| `ipd_feat` | `[B, 24, T, 32]` | 12 个麦对的 cos/sin 相位差 |
| `srp_map` | `[B, 1, T, 72]` | 72 个方位 bin 的 SRP-PHAT 空间谱 |
| `vad_ratio` | `[B, T, 1]` | 语音活动比例 |

这里不是纯 TDOA 反解，而是：

```text
多通道波形
  -> STFT
  -> logmel / IPD / SRP-PHAT / VAD
  -> 深度模型学习当前方位 heatmap
```

## 4. 当前模型主干

主模型位于：

[tracktrend_net.py](/c:/Users/haoming%20lan/Desktop/dc/src/uca8/models/tracktrend_net.py)

模型名：

```text
UCA8TrackTrendNet
```

整体流程：

```text
8ch waveform
  -> UCAFeatureFrontend
       -> logmel_ref / logmel_rms
       -> ipd_feat
       -> srp_map
       -> vad_ratio

  -> SpectrogramEncoder
  -> IPDEncoder
  -> SRPEncoder

  -> concat
  -> Linear + LayerNorm + GELU
  -> CausalTCN

  -> heatmap_logits
  -> count_logits / slot_logits / future heads / motion head
```

当前主实验只使用：

```text
heatmap_logits
```

其他输出头保留在代码里，但在 localization-only 配置中不作为有效实验结论。

## 5. 三路编码器

编码器位于：

[spec_encoder.py](/c:/Users/haoming%20lan/Desktop/dc/src/uca8/models/spec_encoder.py)  
[spatial_encoder.py](/c:/Users/haoming%20lan/Desktop/dc/src/uca8/models/spatial_encoder.py)

### 5.1 SpectrogramEncoder

输入：

```text
logmel_ref + logmel_rms
```

输出：

```text
h_spec: [B, T, 128]
```

作用是学习语音内容、能量变化和语音活动相关模式。

### 5.2 IPDEncoder

输入：

```text
ipd_feat: [B, 24, T, 32]
```

输出：

```text
h_ipd: [B, T, 64]
```

作用是学习通道间相位差和方位之间的关系。

### 5.3 SRPEncoder

输入：

```text
srp_map: [B, 1, T, 72]
```

输出：

```text
h_srp: [B, T, 64]
```

这里是本次架构修正的关键点。

旧实现把 SRP 的方位维直接平均：

```text
conv(srp_map).mean(dim=-1)
```

这会抹掉“峰值在哪个方位 bin”这一核心信息，导致模型虽然看到了 SRP，但学不到方向位置。

新实现保留 72 个 azimuth bin：

```text
conv(srp_map)
  -> [B, 32, T, 72]
  -> permute
  -> flatten 32 * 72
  -> Linear
  -> [B, T, 64]
```

对应代码：

[spatial_encoder.py](/c:/Users/haoming%20lan/Desktop/dc/src/uca8/models/spatial_encoder.py:29)

这次修正后，当前定位指标出现了数量级提升。

## 10. 和 RealMAN baseline 的关系

RealMAN 官方定位 baseline 主要是：

```text
当前 azimuth localization
指标：MAE [deg], ACC(5deg)
```

我们现在的 `heatmap_logits -> argmax` 是最接近官方 spatial spectrum 输出的部分。

但必须注明两点：

1. 当前使用 `CH9-CH16 ring2_8ch` 子阵列，官方默认 CRNN/IPDnet 通道不是这组。
2. 当前训练是短跑调试，不是完整 paper-grade 训练 schedule。

因此论文里更稳妥的写法是：

```text
在 RealMAN ring2_8ch 子阵列设置下，当前方法取得了接近官方定位 baseline 量级的方位定位结果。
```

不要写成：

```text
严格复现并超过 RealMAN 官方 baseline。
```

## 11. 当前风险和下一步

当前还存在这些风险：

1. 训练和评估协议还没有完全对齐官方 CRNN/IPDnet 默认设置。
2. `slot/count/future/trend` 目前只是保留接口，不能作为主结果。
3. moving 子集仍明显弱于 static，后续要检查标签时间对齐和运动窗口策略。
4. 当前只跑了 4096 samples / 480 steps，需要全量数据后再做正式结果。
5. 如果要严格对标官方表格，需要改用官方默认通道或明确训练一个对应通道版本。

下一步建议：

```text
先固定当前定位主线
-> 做 8ch ring2 更长训练
-> 做 SRP/IPD/logmel 消融
-> 全量数据到位后跑正式 val/test
-> 再决定是否恢复 count / slot / future
```

## 12. 最终架构总结

当前可用的核心架构是：

```text
RealMAN CH9-CH16 ring2_8ch
  -> logmel / IPD / SRP-PHAT / VAD
  -> 三路编码器
       SpectrogramEncoder
       IPDEncoder
       SRPEncoder 保留 azimuth bin
  -> Causal TCN
  -> current heatmap_logits
  -> MAE / ACC@5 评估
```

这条线已经形成可解释、可训练、可和 RealMAN 定位任务对话的最小闭环。  
多任务跟踪和未来趋势预测暂时作为扩展模块，而不是当前论文结果的主支撑。
